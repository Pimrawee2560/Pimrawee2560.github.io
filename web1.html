Content Management System and Template

For this project, I've taken on a full-stack role with a heavier emphasis on the backend. Since it's a paired project, we needed to divide the responsibilities, with my partner focusing on the frontend. However, if my partner isn't able to manage, I will step in to assist. This project is a website focused on content management and website administration, allowing users to easily add, delete, and edit website content. It offers a variety of website content templates that most websites typically require, providing web developers with convenient options. This enables them to choose a format or layout that beautifully suits the theme or style of the website. The system offers flexibility for editing and customization according to user needs, without requiring specialized technical knowledge.

The system should have the following capabilities:

  • The ability to create and edit website content without needing to know HTML or CSS coding.
  • The ability to create and duplicate entire websites.
  • The option to choose from various templates.
  • Support for creating multiple user account types, such as administrators, authors, and general users, along with an access control system to manage different permissions.
  • And it will have other capabilities as well.

Tools:

  • Frontend: HTML, CSS, JavaScript
  • Backend: PHP (for server-side logic)
  • Database: MySQL (for storing and managing data)
  • Database Management: phpMyAdmin (a tool for interacting with the MySQL database)
  • Design Tool: Figma (used for creating the website's visual design)
  • Tool: XAMPP (Simulate a server on a computer),Apache(Is a web server),TinyMCE (Text Editor), Visual Studio Code
